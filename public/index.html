Pasos de uso

  1. Configura variables en el entorno de ejecución: TWITCH_CLIENT_ID y TWITCH_CLIENT_SECRET.
  2. Arranca el servidor: npm install y npm start.
  3. Usa el endpoint:
    • /api/followage?channel=<canal>&format=text|json&lang=es|en (requiere sesión; usa tu usuario por defecto)
  4. Salud del servicio: /health.
  5. El campo Usuario se completa con tu sesión y se bloquea.

Modo público: si el canal/moderador está autenticado, puedes consultar cualquier usuario sin iniciar sesión de usuario.

Command APIs

Patrón de URL compatible con Nightbot y otros bots:

GET /twitch/followage/{StreamerUsername}/{ViewerUsername}?format=ymdhis|json&ping=true|false&moderatorId={ModeratorId}&lang=es|en&token={OAuthMod}
  • StreamerUsername: login del canal.
  • ViewerUsername: login del usuario.
  • format: ymdhis (por defecto) o json.
  • ping: añade menciones en texto (true|false).
  • moderatorId: opcional, id del moderador para seleccionar token cacheado si existe.
  • lang: es o en.
  • token: opcional; OAuth del canal/mod con moderator:read:followers. Si lo pasas, no dependes de memoria/cookies/env.

Ejemplos Nightbot (producción):

!commands add -ul=everyone !followage $(urlfetch https://followage-api.onrender.com/twitch/followage/$(channel)/$(user)?format=ymdhis&lang=es&token=OAUTH_DEL_MOD)
!commands add -ul=everyone !followage $(urlfetch https://followage-api.onrender.com/twitch/followage/$(channel)/$(touser)?format=json&lang=en&ping=true&token=OAUTH_DEL_MOD)

Aviso: incluir token en la URL lo expone a quien vea el comando; usa un token dedicado/rotatorio para el bot.

Notas

  • Token de aplicación de Twitch cacheado y renovado automáticamente.
  • Duraciones aproximadas para meses/días (30 días/mes, 365 días/año).
  • Si necesitas precisión de calendario exacta, se puede ajustar.
